JsonSchema框架对比

React-JsonSchema-Form

**React-JsonSchema-Form **是一个基于 React 的库，可使用 JSON Schema 描述表单结构并生成交互式表单。它支持大量的表单组件（例如文本框、下拉列表、日期选择器等），可以灵活地定制表单组件的外观和行为，同时提供了钩子函数用于处理表单数据和验证等操作。React-JsonSchema-Form 旨在简化开发人员构建表单的过程，同时提高表单的可维护性。
官网：https://github.com/rjsf-team/react-jsonschema-form

Formily2

Formily2 是一款基于 React 的数据驱动表单解决方案，它能够轻松地构建复杂的表单，同时提供了丰富的表单组件和高度可配置的表单验证规则。Formily2 还支持表单布局和样式定制，使表单能够适应不同的 UI 需求。除此之外，Formily2 提供了多种数据源（例如 Redux、MobX、React Context 等）以及丰富的 API，可以满足各种场景下的表单数据管理需求。

对比

React-JsonSchema-Form 与 Formily2 都是基于 React 的表单库，它们都支持使用 JSON Schema 描述表单结构，同时提供了丰富的表单组件和可定制的表单验证规则。但是，两者在设计理念和功能上存在差异：

  • React-JsonSchema-Form 旨在简化表单的构建过程，它提供了一种通过 JSON Schema 描述表单结构并自动生成表单的方式，以及方便的钩子函数处理表单数据和验证等操作。相比之下，Formily2 更注重数据驱动，它提供了丰富的数据管理和状态管理功能，以及更灵活的表单组件和表单布局方式。
  • React-JsonSchema-Form 的定制能力相对较弱，它提供了一些简单的样式配置和组件属性配置。Formily2 则提供了更丰富的表单样式和布局定制方式，以及可扩展的表单组件和验证规则。
  • React-JsonSchema-Form 的文档和社区相对较为成熟和活跃，同时它也有一些比较成熟的扩展库和示例项目。Formily2 的文档和社区尚不如 React-JsonSchema-Form 成熟，但它有着活跃的开发团队和较为丰富的功能。

    综上所述，React-JsonSchema-Form 适用于简单的表单场景，可以快速构建简单的表单，并提供简单的数据处理和验证功能。Formily2 则适用于更复杂的表单场景，需要较为丰富的数据。

官网：https://v2.formilyjs.org/zh-CN/guide

组件通信

在React中，组件之间的通信可以通过Props、Context、Refs、事件订阅发布等方式来实现。下面是一些具体的方法：

  1. Props：父组件可以通过Props将数据传递给子组件，从而实现通信。子组件可以通过props接收父组件传递过来的数据。
  2. Context：Context可以将数据传递给所有的子组件，无论子组件的嵌套层级有多深。通过创建Context对象，可以将数据传递给Context.Provider组件，在子组件中可以通过Context.Consumer来接收数据。
  3. Refs：通过Refs，可以在组件之间传递引用，从而实现通信。通过在组件中创建Refs，可以获取到组件的实例，从而可以调用组件的方法或访问组件的属性。
  4. 事件订阅发布：组件可以通过自定义事件的方式来实现通信。可以通过一个事件订阅者来订阅某个事件，当事件被触发时，所有订阅该事件的组件都会接收到通知。
  5. 全局状态管理库：例如Redux、MobX等，通过使用这些全局状态管理库，可以将应用程序的状态保存在全局的store中，不同的组件可以通过dispatch action的方式来更新状态，并通过订阅store来获取状态的变化。
  6. Recoil：Recoil是一个状态管理库，用于管理React应用程序中的全局状态。它是由Facebook开发的开源库，旨在解决React应用程序中状态管理的问题。Recoil提供了一种简单、直观的方式来管理全局状态，它的核心思想是“原子状态”，即将全局状态拆分成多个独立的小状态，每个小状态可以单独管理和使用，从而实现更好的可维护性和可重用性。

    总之，React中有多种方法可以实现组件之间的通信，具体的实现方式取决于应用场景和个人偏好。
